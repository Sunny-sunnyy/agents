Section 2

Cheat Sheet - Tóm tắt nhanh

Core idea - Ý chính

Hệ sinh thái LangChain gồm 4 lớp abstraction (trừu tượng hóa) xếp chồng lên nhau. Mỗi lớp xây dựng trên lớp bên dưới — càng lên cao, bạn viết ít code hơn nhưng trao nhiều quyền kiểm soát hơn cho framework. Tất cả đều dùng chung phương thức invoke().

Must know - Cần nhớ

  • Layer 1 (langchain-core): ChatOpenAI, @tool, tool loop thủ công
  • Layer 2 (langgraph): StateGraph, nodes, edges, conditional routing
  • Layer 3 (langchain): create_agent() tạo sẵn agent loop
  • Layer 4 (deepagents): create_deep_agent() — planning, filesystem, sub-agents
  • create_agent bên trong thực chất tạo một LangGraph graph — chính là graph ta build thủ công ở Layer 2

Use in practice - Dùng khi nào

Khi thiết kế hệ thống agent thực tế, bạn cần chọn đúng tầng:

  • Cần kiểm soát chi tiết từng bước? → Layer 1-2
  • Cần agent loop nhanh, common pattern? → Layer 3
  • Cần agent tự lên kế hoạch, viết file, tạo sub-agent? → Layer 4
  • Ứng dụng thực tế thường mix nhiều layer cùng lúc

Section 4

Core Theory - Lý thuyết cốt lõi

Transcript Slide Code

Key concepts - Khái niệm chính

  • Abstraction layer - lớp trừu tượng hóa: Mỗi layer ẩn đi chi tiết phức tạp của layer bên dưới, cho phép developer làm việc ở mức cao hơn.
  • Building blocks - khối xây dựng cơ bản: langchain-core + langchain-openai. Bao gồm ChatOpenAI (wrap API call), @tool decorator (tạo tool từ Python function), và message types (HumanMessage, ToolMessage).
  • Orchestration - điều phối: langgraph mô hình hóa workflow thành dependency graph (đồ thị phụ thuộc) với nodes (đơn vị công việc) và edges (kết nối giữa các nodes).
  • Agent framework - khung agent: create_agent() trong package langchain. Tạo sẵn toàn bộ agent loop (vòng lặp agent) — bên trong nó tạo LangGraph graph giống hệt graph ta build thủ công.
  • Agent harness - khung vận hành agent: create_deep_agent() từ package deepagents. Bọc thêm planning (lập kế hoạch), virtual filesystem (hệ thống file ảo), và sub-agents (agent con).

Concept relationships - Quan hệ giữa khái niệm

Quan hệ xếp chồng (layered dependency):

  • Layer 4 dùng Layer 3 bên dưới
  • Layer 3 tạo ra một Layer 2 graph
  • Layer 2 gọi các components từ Layer 1

Trade-off cốt lõi: càng lên cao → viết ít code hơn, nhưng ít kiểm soát hơn. Đây là control vs. convenience trade-off — giống nguyên lý chung trong software engineering.

Điểm gây nhầm lẫn: Layer 1 và Layer 3 đều thuộc package langchain, trong khi Layer 2 thuộc package langgraph riêng biệt.

Deep explanation - Chi tiết 4 layers

Layer 1: Building Blocks

Package: langchain-core, langchain-openai

Đây là nơi LangChain bắt đầu (cuối 2022). Cung cấp:

  • ChatOpenAI: Wrap API call đến OpenAI
  • @tool decorator: Biến Python function thành tool cho LLM. Docstring tự động thành description, type hints thành argument schema.
  • bind_tools(): Gắn tools vào LLM
  • Tool loop phải viết thủ công: gọi LLM → nhận tool_calls → thực thi tool → gửi ToolMessage → gọi LLM lần 2

Layer này cho bạn toàn quyền kiểm soát nhưng phải viết nhiều code nhất.

Layer 2: Orchestration (LangGraph)

Package: langgraph

Mô hình hóa workflow thành dependency graph:

  • StateGraph: Định nghĩa graph với shared state
  • Nodes: Python functions đóng vai trò đơn vị công việc
  • Edges: Kết nối giữa nodes (có thể conditional)
  • ToolNode: Node prebuilt chạy tool
  • tools_condition: Routing có điều kiện — nếu LLM muốn gọi tool thì route đến ToolNode, nếu không thì kết thúc
  • Recipe: define state → add nodes → add edges → compile → invoke

Layer 3: Agent (create_agent)

Package: langchain

Wrapper tạo sẵn toàn bộ agent loop:

  • Input: model, tools, system_prompt
  • Output: Compiled LangGraph graph (có thể get_graph().draw_mermaid_png() giống hệt graph Layer 2)
  • Tương tự OpenAI Agents SDK trong hệ sinh thái LangChain
  • Key insight: Không có "magic" nào — nó chỉ là syntactic sugar (cú pháp tiện lợi) trên Layer 2

Layer 4: Harness (Deep Agents)

Package: deepagents

Agent harness cho multi-step work:

  • Planning tool: Agent tự viết todo list cho mình
  • Virtual filesystem: Đọc/ghi file trong sandbox
  • Sub-agents: Ủy thác nhiệm vụ cho agent con
  • Agent chạy tự chủ (autonomous) — thứ tự tool calls có thể thay đổi mỗi lần chạy
  • Tương tự Claude Agent SDK hoặc OpenAI sandbox agents
Tại sao Layer 1 & 3 đều là LangChain? Kiến thức mở rộng

Đây là nguồn gốc sự nhầm lẫn phổ biến nhất. langchain-core (Layer 1) ra đời trước — là abstraction thuần túy cho LLM calls. Sau đó langgraph (Layer 2) được tách thành package riêng. Cuối cùng, create_agent() (Layer 3) được thêm lại vào package langchain chính — tạo ra cấu trúc "sandwich" khó hiểu: LangChain → LangGraph → LangChain.

Hiểu rõ sự phân chia này giúp bạn đọc docs và imports đúng package.

Section 5

Workflow / Pipeline - Quy trình / luồng hoạt động

Transcript Code

High-level flow - Luồng tổng quát

Cùng một task (tra giá cổ phiếu) được thực hiện qua 4 layers với mức độ abstraction tăng dần:

  1. Layer 1: Gọi llm.invoke() → nhận tool_calls → chạy tool thủ công → gửi ToolMessage → gọi invoke() lần 2 để lấy câu trả lời cuối cùng
  2. Layer 2: Define StateGraph với chatbot node + ToolNode + conditional edges → graph.invoke() chạy toàn bộ loop tự động
  3. Layer 3: create_agent(model, tools, prompt) → agent.invoke() — không cần viết graph code
  4. Layer 4: create_deep_agent(model, tools, prompt) → deep_agent.invoke() — agent tự plan, gọi tool, viết file

Data / control flow - Luồng dữ liệu / điều khiển

Input: Câu hỏi của user (ví dụ "What is the share price of Amazon?")

Qua mỗi layer:

  • Layer 1: User ↔ Code ↔ LLM ↔ Tool — tất cả do developer điều phối
  • Layer 2: User → Graph → (LLM node ↔ Tool node) → Output — graph engine điều phối
  • Layer 3: User → Agent → Output — agent loop tự điều phối (bên trong là graph Layer 2)
  • Layer 4: User → Deep Agent → (plan → tools → file writes) → Output — harness tự quản lý lifecycle

Output: Câu trả lời cuối cùng (Layer 1-3) hoặc file artifact (Layer 4)

invoke() - Phương thức thống nhất

invoke() là phương thức chung mà bạn gọi ở mọi layer. Dù là llm.invoke(), graph.invoke(), agent.invoke(), hay deep_agent.invoke() — cùng một interface. Đây là design principle quan trọng: uniform invocation (giao diện gọi thống nhất) giúp chuyển đổi giữa các layers dễ dàng.

Section 6

Techniques - Kỹ thuật sử dụng

@tool Decorator

Purpose: Biến Python function thành tool mà LLM có thể gọi. Docstring tự động thành JSON schema — không cần viết tay.

When to use: Khi cần cho LLM truy cập external data hoặc thực hiện action (API call, database query, file I/O).

Trade-off: Docstring phải chính xác và rõ ràng — LLM dựa vào đó để quyết định khi nào và cách gọi tool.

bind_tools() + Tool Loop

Purpose: Gắn danh sách tools vào LLM object. Ở Layer 1, bạn phải tự viết tool loop (vòng lặp thực thi tool).

When to use: Khi cần kiểm soát chi tiết: quyết định có thực thi tool call không, xử lý error riêng, log từng bước.

Trade-off: Viết nhiều code hơn nhưng kiểm soát hoàn toàn. Layer 2+ automate phần này.

StateGraph Pattern

Purpose: Mô hình hóa workflow thành directed graph có state. ToolNode + tools_condition là prebuilt nodes phổ biến nhất.

When to use: Workflow cần routing có điều kiện, nhiều bước lặp lại, hoặc custom logic giữa các bước.

Trade-off: Powerful nhưng verbose hơn Layer 3. Đổi lại, bạn có thể thiết kế graph bất kỳ — không bị giới hạn bởi prebuilt pattern.

create_agent() Shortcut

Purpose: Tạo standard agent loop (LLM + tools) bằng một lệnh duy nhất. Trả về compiled LangGraph graph.

When to use: Agent loop đơn giản (LLM gọi tools trong vòng lặp) — không cần custom routing hay state phức tạp.

Trade-off: Nhanh nhưng ít linh hoạt. Nếu cần Worker-Evaluator pattern (Day 4), phải quay lại Layer 2.

create_deep_agent() Harness

Purpose: Agent tự chủ có built-in planning (todo tool), virtual filesystem, và sub-agents.

When to use: Multi-step tasks yêu cầu agent tự lên kế hoạch, lưu kết quả, ủy thác sub-tasks.

Trade-off: Ít kiểm soát nhất. Agent có thể gọi tools theo thứ tự khác nhau mỗi lần chạy (non-deterministic).

Graph Inspection Kiến thức mở rộng

Purpose: Dùng agent.get_graph().draw_mermaid_png() để visualize graph bên trong bất kỳ agent nào.

When to use: Debug, hiểu cấu trúc bên trong, hoặc chứng minh rằng create_agent thực chất là LangGraph.

Trade-off: Không ảnh hưởng runtime — chỉ là công cụ inspection.

Tool Definition - Định nghĩa tool

Hàm get_share_price dùng @tool decorator. Docstring tự động thành tool description cho LLM.

Layer: 1 - Building Blocks Key: @tool decorator, docstring-as-schema 
# Hàm helper gọi Alpha Vantage API lấy giá real-time
def fetch_live_price(symbol: str) -> float:
    response = requests.get("https://www.alphavantage.co/query",
        timeout=10,
        params={"function": "GLOBAL_QUOTE",
                "symbol": symbol,
                "apikey": os.environ["ALPHAVANTAGE_API_KEY"]})
    return float(response.json()["Global Quote"]["05. price"])

# @tool decorator: biến function thành tool cho LLM
# Docstring "Return the current share price..." tự động thành JSON schema
@tool
def get_share_price(symbol: str) -> float:
    """Return the current share price for a given ticker symbol."""
    if os.environ.get("ALPHAVANTAGE_API_KEY"):
        try:
            return fetch_live_price(symbol)
        except Exception:
            pass
    # Fallback: bảng giá giả nếu không có API key
    fake_prices = {"AAPL": 241.5, "GOOG": 168.2, "AMZN": 198.0}
    return fake_prices.get(symbol.upper(), 0.0)

Layer 1: Manual Tool Loop - Vòng lặp tool thủ công

Ở Layer 1, developer phải tự xử lý: gọi LLM → đọc tool_calls → chạy tool → gửi kết quả lại → gọi LLM lần 2.

Layer: 1 - Building Blocks Key: bind_tools, tool_calls, ToolMessage, manual invoke loop 
# Bước 1: Gắn tool vào LLM
llm_with_tools = llm.bind_tools([get_share_price])

# Bước 2: Gọi LLM - nó sẽ trả về tool_calls, KHÔNG phải câu trả lời
conversation = [HumanMessage("What is the share price of Amazon?")]
ai_message = llm_with_tools.invoke(conversation)
conversation.append(ai_message)

# Bước 3: Developer tự chạy tool và gửi kết quả lại
for call in ai_message.tool_calls:
    result = get_share_price.invoke(call["args"])
    conversation.append(
        ToolMessage(content=str(result), tool_call_id=call["id"])
    )

# Bước 4: Gọi LLM lần 2 để lấy câu trả lời cuối cùng
print(llm_with_tools.invoke(conversation).content)
# Output: "Amazon is currently priced at $238."

Layer 2: LangGraph Orchestration - Điều phối bằng đồ thị

Cùng logic nhưng được mô hình hóa thành dependency graph. Graph engine tự động chạy tool loop.

Layer: 2 - Orchestration Key: StateGraph, ToolNode, tools_condition, compile 
# Định nghĩa State với messages list (quen thuộc từ Day 1-2)
class State(TypedDict):
    messages: Annotated[list, add_messages]

# Node "chatbot": gọi LLM với tools
def chatbot(state: State) -> dict:
    return {"messages": [llm_with_tools.invoke(state["messages"])]}

# Xây dựng graph: recipe chuẩn define-add-compile
builder = StateGraph(State)
builder.add_node("chatbot", chatbot)           # Node chính
builder.add_node("tools", ToolNode([get_share_price]))  # Node chạy tool
builder.add_edge(START, "chatbot")              # Bắt đầu từ chatbot
builder.add_conditional_edges("chatbot", tools_condition)  # Routing có điều kiện
builder.add_edge("tools", "chatbot")            # Sau tool, quay lại chatbot
graph = builder.compile()

# Chạy graph - tự động xử lý tool loop!
result = graph.invoke({"messages": [{"role": "user",
    "content": "Which is more expensive, Apple or Google stock?"}]})
print(result["messages"][-1].content)

Layer 3: create_agent() - Agent chỉ bằng 1 lệnh

Tất cả graph code bên trên được rút gọn thành create_agent(). Kết quả là compiled LangGraph graph — y hệt graph Layer 2.

Layer: 3 - Agent Key: create_agent, model + tools + prompt 
# Tạo agent chỉ với model, tools, prompt - không cần graph code
agent = create_agent(
    model="openai:gpt-5.4-mini",
    tools=[get_share_price],
    system_prompt="You are a helpful financial assistant. Use your tools.",
)

# Gọi agent - cùng invoke() interface
result = agent.invoke({"messages": [{"role": "user",
    "content": "What are the share prices of Apple, Google and Amazon?"}]})
print(result["messages"][-1].content)

# THE REVEAL: Graph bên trong giống hệt graph ta build thủ công ở Layer 2!
display(Image(agent.get_graph().draw_mermaid_png()))

Layer 4: create_deep_agent() - Agent Harness tự chủ

Agent tự lên kế hoạch (todo tool), gọi tools, viết kết quả ra file. Thứ tự tool calls có thể khác nhau mỗi lần chạy.

Layer: 4 - Harness Key: create_deep_agent, write_todos, write_file, virtual filesystem 
# Tạo deep agent - có thêm built-in tools (todo, filesystem)
deep_agent = create_deep_agent(
    model="openai:gpt-5.4-mini",
    tools=[get_share_price],
    system_prompt="You are an analyst. Plan your work with your todo tool, "
                  "use your tools, and write your answer to a file.",
)

# Giao nhiệm vụ phức tạp: tra giá + xếp hạng + viết file
result = deep_agent.invoke({"messages": [{"role": "user",
    "content": "Look up the share prices of AAPL, GOOG and AMZN, "
              "then write a short markdown note ranking them to prices.md"}]})

# Agent tự gọi: write_todos -> get_share_price x3 -> write_file
tools_used = [tc["name"] for m in result["messages"]
              for tc in (getattr(m, "tool_calls", []) or [])]
print("Tools called:", tools_used)

# Đọc file agent đã viết trong virtual filesystem
display(Markdown(result["files"]["/prices.md"]["content"]))

Section 9

Pitfalls - Lỗi / bẫy thường gặp

Failure modes - Trường hợp dễ fail

  • Nhầm package: Import create_agent từ langchain (đúng) vs. langchain-core (sai). Layer 1 và 3 đều là "LangChain" nhưng khác package.
  • Quên tool loop ở Layer 1: Gọi llm_with_tools.invoke() chỉ trả về tool_calls, không tự động chạy tool. Phải tự thêm ToolMessage và invoke lần 2.
  • Dùng Layer 3-4 cho custom workflow: create_agent chỉ tạo standard loop. Nếu cần Worker-Evaluator pattern hoặc multi-branch routing, phải quay về Layer 2.
  • Non-determinism ở Layer 4: Deep agent có thể gọi tools theo thứ tự khác nhau mỗi lần chạy. Viết test khó hơn.
  • Docstring tool kém: LLM dựa vào docstring để hiểu khi nào gọi tool. Docstring mập mờ = tool không được gọi đúng lúc.

Fix / prevention - Cách tránh

  • Kiểm tra import path: from langchain.agents import create_agent (Layer 3), from langchain_core.tools import tool (Layer 1), from langgraph.graph import StateGraph (Layer 2).
  • Dùng graph inspection: agent.get_graph().draw_mermaid_png() để verify cấu trúc graph bên trong.
  • Chọn đúng layer: Bắt đầu từ Layer 3, nếu thiếu linh hoạt thì drop xuống Layer 2. Không cố ép workflow phức tạp vào Layer 3-4.
  • Test Layer 4 bằng output validation: Thay vì test thứ tự tool calls, test nội dung file output hoặc kết quả cuối cùng.
  • Viết docstring tool rõ ràng: Mô tả chính xác tool làm gì, input/output là gì. Coi docstring như API contract.
Section 10

Flashcards / Q&A - Thẻ ôn tập

Q: Hệ sinh thái LangChain có bao nhiêu lớp abstraction?

A: 4 lớp: (1) Building Blocks (langchain-core), (2) Orchestration (langgraph), (3) Agent (langchain - create_agent), (4) Harness (deepagents - create_deep_agent).

Q: create_agent() bên trong thực chất làm gì?

A: Tạo một compiled LangGraph graph (Layer 2) với chatbot node + ToolNode + conditional edges. Có thể verify bằng agent.get_graph().draw_mermaid_png() — graph giống hệt graph xây thủ công.

Q: Tại sao Layer 1 và Layer 3 đều gọi là "LangChain"?

A: Vì lý do lịch sử: langchain-core (Layer 1) ra đời trước là abstraction cơ bản. Sau đó langgraph (Layer 2) tách riêng. Cuối cùng create_agent() (Layer 3) được thêm vào package langchain chính, tạo cấu trúc "sandwich" gây nhầm lẫn.

Q: Deep Agents (Layer 4) có gì mà Layer 3 không có?

A: 3 built-in capabilities: (1) Planning tool (tự viết todo list), (2) Virtual filesystem (đọc/ghi file), (3) Sub-agents (ủy thác cho agent con). Agent hoạt động tự chủ — tự quyết thứ tự và số lần gọi tools.

Q: Ở Layer 1, sau khi gọi llm_with_tools.invoke(), bạn nhận được gì?

A: Nhận được AI message chứa tool_calls (danh sách tool mà LLM muốn gọi), không phải câu trả lời cuối cùng. Phải tự chạy tool, wrap kết quả trong ToolMessage, rồi invoke lần 2 để LLM trả lời.

Quick Memory - Ôn nhanh cuối bài

Must remember - Phải nhớ

  1. 4 layers: Building Blocks → Orchestration → Agent → Harness. Mỗi layer xây trên layer dưới.
  2. Control vs. Convenience: Càng lên cao, viết ít code hơn nhưng ít kiểm soát hơn. Ứng dụng thực tế mix nhiều layers.
  3. invoke() là phương thức thống nhất xuyên suốt mọi layer.
  4. create_agent() chỉ là syntactic sugar — bên trong tạo LangGraph graph y hệt graph build thủ công.
  5. Deep Agents có planning tool, virtual filesystem, sub-agents — agent tự chủ nhưng non-deterministic.
  6. Layer 1 & 3 đều là "LangChain" nhưng Layer 2 là "LangGraph" — nguồn gốc sự nhầm lẫn phổ biến.
